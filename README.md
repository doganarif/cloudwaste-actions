# CloudWaste GitHub Actions — Cost Preview

Get automated cost estimates on every pull request that changes infrastructure code.

When a PR modifies `.tf` files, this workflow comments with the estimated monthly cost of your AWS resources.

## Setup

### 1. Get your API token

Sign up at [cloudwaste.io](https://cloudwaste.io), then go to **Settings → Autopilot** to find your API token.

### 2. Add the secret to your repository

Go to **Settings → Secrets and variables → Actions → New repository secret**.

- Name: `CLOUDWASTE_API_TOKEN`
- Value: your API token from CloudWaste

### 3. Add the workflow file

Copy `.github/workflows/cost-preview.yml` from this repository into your own repo at the same path.

Or run:

```bash
mkdir -p .github/workflows
curl -sL https://raw.githubusercontent.com/doganarif/cloudwaste-actions/main/.github/workflows/cost-preview.yml \
  -o .github/workflows/cost-preview.yml
```

### 4. Open a pull request

Create or update a PR that changes `.tf`, `.tfvars`, or `docker-compose*.yml` files. CloudWaste will post a comment like:

```
## CloudWaste Cost Preview

**Estimated monthly cost:** $284.40

| Instance Type | Count | Monthly Cost |
|:-------------|------:|-------------:|
| t3.medium    |     1 |       $30.37 |
| m5.large     |     1 |       $69.12 |
| db.t3.medium |     1 |      $184.91 |

*Generated by [CloudWaste](https://cloudwaste.io)*
```

## How it works

1. The workflow triggers on PRs that touch Terraform files
2. It parses `instance_type` and `instance_class` from `.tf` files
3. Sends the extracted resources to the CloudWaste API (`/api/cost-preview`)
4. Posts the cost breakdown as a PR comment (updates existing comment on re-runs)

## Configuration

| Variable | Default | Description |
|---|---|---|
| `CLOUDWASTE_API_TOKEN` (secret) | — | Your CloudWaste API token |
| `CLOUDWASTE_URL` (variable) | `https://cloudwaste.io` | CloudWaste API base URL |
